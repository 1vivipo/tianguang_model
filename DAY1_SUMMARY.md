## 一、项目背景

### 1.1 项目目标
构建一个真正能自主思考、自主学习、自主进化的AI系统。

### 1.2 核心理念
- **体积可以小，逻辑必须硬**
- **见识可以少，判断必须准**
- **话可以不多，每句都要解决问题**
- **永远把「怎么思考」放在「知道什么」前面**

## 二、架构设计

### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    天光模型架构                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   【知识层】外部知识库                                  │
│   ├── 天光 (MetaBrain) - 检索+模板                     │
│   ├── 洱海 (Urhai) - 神经网络+学习                     │
│   ├── 突突 (Tutu) - 自动爬取+自动学习                  │
│   └── 黑洞 (Blackhole) - 100只虫子并行爬取             │
│                                                         │
│   【核心层】真正的大脑                                  │
│   └── 核心大脑 (Core Brain) - 思考、推理、决策         │
│                                                         │
│   【集成层】智脑                                        │
│   └── 100只小脑合并 = 真正的大模型                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.2 核心大脑架构

```
【感官层】输入（暂缓，需传感器）
├── 视觉脑 - 看到图像
├── 听觉脑 - 听到声音
├── 语言脑 - 读到文本
└── 触觉脑 - 感知数据

【理解层】读懂
├── 图像理解脑 - 看懂图像内容
├── 声音理解脑 - 听懂语音含义
├── 文本理解脑 - 读懂文字意思
└── 情感理解脑 - 感知情绪

【思考层】处理（核心）
├── 记忆脑 - 调用知识
├── 推理脑 - 逻辑推理
├── 分析脑 - 拆解问题
├── 判断脑 - 做出判断
└── 创意脑 - 生成想法

【解决层】输出
├── 决策脑 - 给出方案
├── 表达脑 - 组织语言
├── 行动脑 - 执行任务
└── 学习脑 - 总结经验
```

## 四、技术实现

### 4.1 核心大脑组件

```python
# 思考引擎
class ThinkingEngine:
    """问题拆解 → 定位核心 → 分步解决 → 输出答案"""

# 问题解决器
class ProblemSolver:
    """识别问题类型 → 查找知识 → 给出答案"""

# 对话记忆
class DialogMemory:
    """短期记忆 → 长期记忆 → 经验积累"""

# 核心大脑
class CoreBrain:
    """整合所有组件，实现完整对话"""
```

| 方法类 | 怎么做？ | 给出步骤 |
| 比较类 | 哪个好？ | 对比分析 |
| 判断类 | 是不是？ | 给出判断 |

## 七、关键决策

### 7.1 为什么核心大脑是关键？

```
天光/洱海/突突/黑洞 = 知识库/配件/工具
核心大脑 = 真正的大脑/底座
100只核心大脑合并 = 真正的大模型
```

### 7.2 为什么先训练思考类？

1. 思考是AI的核心能力
2. 不依赖传感器
3. 可以立即使用
4. 是其他能力的基础

### 7.3 为什么不赶任务？

> **不能赶任务，要打好基础，要真的训好**

每个小脑都要确保能用，再继续下一个。
